# Mobile App Scaffold

## Responsibilities
- Capture or upload meal photos.
- Give users basic image guidance before submit.
- Send image plus authenticated user context to the backend.
- Render structured responses with confidence and ambiguity notes.
- Allow retake and optional user correction feedback.

## Phase 1 UX principles
- Keep the flow to one image and one result card.
- Show bounded language such as “estimated saturated fat.”
- When confidence is low, prioritize clarity over precision.
- Do not render medical recommendations or goal-setting advice in the client.

## Suggested screens
1. Meal capture / upload
2. Result view with food items, sat-fat estimate, daily total, and disclaimer
3. Retake prompt for bad images
4. Optional feedback / correction modal
